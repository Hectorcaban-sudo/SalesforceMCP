# Salesforce MCP Server

A custom Model Context Protocol (MCP) server built with **C# ASP.NET Core** that exposes Salesforce data operations via natural language and SOQL. Includes a Python LangChain agent and chatbot client.

## Architecture

```
SalesforceMcp/
├── src/
│   ├── McpServer/                  # ASP.NET Core 8 MCP Server
│   │   ├── Features/               # Vertical Slice Architecture
│   │   │   ├── Query/              # QueryCommand + Handler + Controller
│   │   │   ├── Create/             # CreateCommand + Handler + Controller
│   │   │   ├── Update/             # UpdateCommand + Handler + Controller
│   │   │   ├── Delete/             # DeleteCommand + Handler + Controller
│   │   │   ├── Export/             # ExportCommand + Handler (CSV)
│   │   │   └── Schema/             # Describe + ListObjects + Reload
│   │   ├── Infrastructure/
│   │   │   ├── Salesforce/         # SalesforceClient, SchemaRegistry, NL Parser
│   │   │   └── Mcp/                # MCP tool registry + endpoint mapping
│   │   ├── Models/                 # Shared domain models
│   │   └── schemas/                # JSON schema files (generated by downloader)
│   └── SchemaDownloader/
│       └── download_schemas.py     # Python script to fetch SF schemas
└── python-clients/
    ├── mcp_tools.py                # LangChain tool wrappers
    ├── salesforce_agent.py         # ReAct agent (autonomous multi-step)
    ├── salesforce_chatbot.py       # Conversational chatbot with memory
    └── requirements.txt
```

## Key Design Decisions

| Decision | Choice | Reason |
|---|---|---|
| Architecture | Vertical Slice | Each feature is self-contained (command + handler + endpoint) |
| Schema | JSON files (no hardcoding) | Schemas generated from live SF org via download script |
| NL Parsing | Custom rule-based | Fast, no external LLM calls needed for parsing |
| CRUD | MediatR commands | Decouples HTTP + MCP from business logic |
| CSV Export | CsvHelper | Handles edge cases (commas, quotes, newlines in values) |

## Step 1: Download Salesforce Schemas

```bash
cd src/SchemaDownloader

# Install dependency
pip install simple-salesforce

# Set credentials
export SF_USERNAME="your@email.com"
export SF_PASSWORD="yourpassword"
export SF_SECURITY_TOKEN="yourtoken"

# Download default set (Account, Contact, Lead, Opportunity, etc.)
python download_schemas.py

# Download ALL objects
python download_schemas.py --all

# Download specific objects
python download_schemas.py --objects Account Contact Opportunity

# Sandbox org
python download_schemas.py --sandbox --all

# Custom output directory
python download_schemas.py --output ./my-schemas --all
```

Copy the generated JSON files to `src/McpServer/schemas/`.

---

## Step 2: Configure the Server

Edit `src/McpServer/appsettings.json`:

```json
{
  "Salesforce": {
    "ClientId": "YOUR_CONNECTED_APP_CLIENT_ID",
    "ClientSecret": "YOUR_CONNECTED_APP_SECRET",
    "Username": "your@email.com",
    "Password": "yourpassword",
    "SecurityToken": "yourtoken",
    "IsSandbox": false
  },
  "Schema": {
    "SchemaDirectory": "./schemas"
  }
}
```

---

## Step 3: Run the Server

```bash
cd src/McpServer
dotnet restore
dotnet run

# Server starts at http://localhost:5000
# MCP endpoint: http://localhost:5000/mcp
# Swagger UI:   http://localhost:5000/swagger
```

---

## Step 4: Run Python Clients

```bash
cd python-clients
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your API keys

# Autonomous agent (good for complex multi-step tasks)
python salesforce_agent.py
python salesforce_agent.py --task "Find all tech companies and export to CSV"
python salesforce_agent.py --provider anthropic

# Conversational chatbot (good for interactive use)
python salesforce_chatbot.py
python salesforce_chatbot.py --provider anthropic
```

## MCP Tools Reference

| Tool | Description |
|---|---|
| `salesforce_query` | Query records with natural language or SOQL |
| `salesforce_create` | Create new records |
| `salesforce_update` | Update existing records by ID |
| `salesforce_delete` | Delete records (requires confirm=true) |
| `salesforce_export_csv` | Export records to CSV file |
| `salesforce_describe` | Show object fields and types |
| `salesforce_list_objects` | List all loaded schemas |

## REST API Endpoints

The server also exposes REST endpoints (in addition to MCP):

```
POST /api/query          - Query records
GET  /api/query?q=...    - Query via GET
POST /api/create         - Create record
PUT  /api/update/{id}    - Update record
DEL  /api/delete/{obj}/{id} - Delete record
GET  /api/export?q=...   - Export CSV download
GET  /api/schema/objects - List objects
GET  /api/schema/objects/{name} - Describe object
POST /api/schema/reload  - Reload schemas from disk
```

## Adding a New Salesforce Object

1. Run the schema downloader for the new object:
   ```bash
   python download_schemas.py --objects MyCustomObject__c
   ```

2. Copy the generated JSON to `src/McpServer/schemas/`

3. Either restart the server OR call `POST /api/schema/reload`

**No code changes needed!** The schema-driven design automatically handles any object.

## Vertical Slice Architecture

Each feature follows this pattern:

```csharp
// Features/Query/QueryFeature.cs

// 1. Command (input DTO)
public class QueryCommand : IRequest<McpResult> { ... }

// 2. Handler (business logic)
public class QueryHandler : IRequestHandler<QueryCommand, McpResult> { ... }

// 3. Controller (HTTP endpoint)
[ApiController] public class QueryController : ControllerBase { ... }
```

The MCP endpoint in `Infrastructure/Mcp/McpToolRegistry.cs` dispatches tool calls to the same MediatR handlers, so HTTP and MCP share identical business logic.
